# ERP Система Московского Зоопарка

## Описание
Консольное приложение для учета животных и инвентаря зоопарка.

## Принципы SOLID в проекте

1. **SRP (Single Responsibility Principle)**:
   - `ZooService` отвечает только за бизнес-логику зоопарка (добавление, подсчеты).
   - `VeterinaryClinic` отвечает только за проверку здоровья.
   - Классы моделей (`Monkey`, `Table`) хранят только данные.

2. **OCP (Open/Closed Principle)**:
   - Иерархия классов `Animal` -> `Herbo`/`Predator` позволяет добавлять новые виды животных, не меняя существующий код сервисов.
   - Мы можем добавить новый тип `Thing`, и он автоматически попадет в инвентаризацию.

3. **LSP (Liskov Substitution Principle)**:
   - Везде, где используется `Animal` или `IInventory`, можно подставить любого наследника (например, `Monkey` или `Table`), и программа будет работать корректно.

4. **ISP (Interface Segregation Principle)**:
   - Интерфейсы разделены: `IAlive` (для тех, кто ест) и `IInventory` (для всего, что на балансе). Столу не нужно реализовывать свойство `Food`.

5. **DIP (Dependency Inversion Principle)**:
   - `ZooService` не зависит от конкретного класса `VeterinaryClinic`. Он зависит от абстракции (интерфейса) `IVeterinaryClinic`. Это позволяет легко подменить клинику (например, на Mock-объект в тестах).
